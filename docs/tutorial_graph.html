In order to build a graph we need to apply a sequence of rules. At first we would use several non-terminal rules building the morphology of the mechanism. First example consists of only two non-terminal rules:

  • FlatCreate - change ROOT for the F node that represents the palm
  • Mount - Change F to F-M-EM which is the palm with one fingertip moved, it is a start for a finger

The terminal rules set the physical properties for the nodes. These properties are determined during terminal node construction above. The non-terminal and terminal rule lists then can be concatenated

rule_action_non_terminal_no_joints = np.asarray(["FlatCreate", "Mount"])

rule_action_terminal_no_joints = np.asarray(
    ["TerminalFlat1", "TerminalTransformL", "TerminalEndLimb"])

rule_action_no_joints = np.r_[rule_action_non_terminal_no_joints, rule_action_terminal_no_joints]

Here we will add simple function that creates a GraphGrammar object and applies rules one by one to get the final graph. Class GraphGrammar represents graphs with additional utility functions, more information in class documentation
This function can be called in other files to create the simplest graph and then use the plot_graph function from the rostok.graph_grammar.node module.

def get_terminal_graph_no_joints():
    G = GraphGrammar()
    for i in list(rule_action_no_joints):
        G.apply_rule(rule_vocab.get_rule(i))
    return G

plot_graph(get_terminal_graph_no_joints())

Lets move to the more complicated sequence of rules that builds mechanism with two fingers. In the non-terminal list we add second Mount rule that adds second finger base two grow finger, and we apply 5 FingerUpper rules each adding a link and joint. Notice, when we have more than one node with the same label in the graph we still do not specify where to apply a rule. Any rule is applied to a node closest to the palm, therefore applying 5 FingerUpper rules we will get one finger with two links and another with three links

To add another finger and get the graph that represents the mechanism with three fingers we add another Mount. here we apply 6 FingerUpper rules, hence we get three fingers with two links in each.

What graphs we can build and their representations as the mechanisms depends entirely on the nodes and rule sets. Each user can create its own nodes/rules in the Rostok framework, but it is also user responsibility to create rules that would lead to physically possible mechanisms. How to use the obtained graphs to start the simulation of the corresponding mechanism one can see in another tutorial
